# LLM Reflective Learning Layer — Design Document

Date: 2026-04-07

## Goal

Add experiential learning to the LLM autonomous trader so it evolves its behavior based on trade outcomes and missed opportunities. The LLM reflects weekly on its experience and produces an evolving "beliefs document" that shapes future decisions. Beliefs persist across sessions.

## Architecture

```
Every bar:
  1. Trade Journal — record entries/exits with thesis + market context
  2. Missed Opportunity Detector — flag big moves on unheld stocks

Every 5 trading days:
  3. Reflection LLM Call — review journal + missed opportunities
     → produce UPDATED BELIEFS (numbered list, max 10)
  4. Beliefs injected into system prompt for next trading period

On session end:
  5. Persist beliefs + journal + missed opps to disk (JSON)
  6. Next session loads and continues from last state
```

## Components

### 1. Enhanced Trade Journal

Current `trade_log` stores basic P&L. Enhanced version stores decision context:

```python
{
    "symbol": "ICICIBANK",
    "action": "BUY",
    "thesis": "THESIS: Banking sector rotation. EVIDENCE: RSI 33... CONVICTION: 8/10",
    "entry_price": 1050.0,
    "entry_date": "2024-03-15",
    "entry_regime": "trending",
    "entry_indicators": {"rsi": 33, "adx": 28, "zscore": -1.8},
    # --- Filled after exit ---
    "exit_price": 1020.0,
    "exit_date": "2024-03-22",
    "pnl_pct": -2.86,
    "outcome": "LOSS",
    "bars_held": 35,
    "post_mortem": "Stop hit. SBIN rallied 5% — wrong stock, right thesis."
}
```

The `post_mortem` is generated by a mini LLM call at trade close (~50 tokens).

### 2. Missed Opportunity Detector

Daily scan of unheld stocks. If stock moved > 2x daily ATR in 5 days:

```python
{
    "symbol": "BHARTIARTL",
    "date": "2024-04-10",
    "move_pct": 7.2,
    "direction": "up",
    "indicators_at_start": {"rsi": 42, "adx": 31, "zscore": -0.3},
    "regime": "trending"
}
```

No LLM call — pure math. Fed into weekly reflection.

### 3. Weekly Reflection Cycle

Separate LLM call every 5 trading days with dedicated reflection prompt:

```
Review your trading performance for the past week.

COMPLETED TRADES: [journal entries with outcomes]
MISSED OPPORTUNITIES: [big moves on unheld stocks]
CURRENT BELIEFS: [existing beliefs]

Update your beliefs. Keep what works. Remove what's contradicted.
Respond as a numbered list (max 10). Be specific:
  Good: "Banking breakouts work in trending regimes — HDFCBANK profitable 3 of 4."
  Bad: "Be more careful." (too vague)
```

Output: 5-10 evolving beliefs stored and injected into trading prompt.

### 4. Beliefs Injection

Trading system prompt gains:

```
YOUR LEARNED BELIEFS (updated weekly from experience):
1. Banking breakouts work in trending regimes.
2. IT stocks move together — if one breaks out, the other follows.
3. Stop-losses at 2x ATR too tight for CNC trades.
...
```

### 5. Persistence

```
strategies/data/llm_experience/
  beliefs.json
  trade_journal.json
  missed_opps.json
  reflection_log.json      # audit trail of all reflections
```

Load on `initialize()`, save on `on_complete()`.
Config: `experience_dir`, `reset_experience: true` to start fresh.

## New Module: `experience_manager.py`

```python
class ExperienceManager:
    def __init__(self, experience_dir, reflection_interval=5)
    def record_entry(self, symbol, action, thesis, indicators, price, date, regime)
    def record_exit(self, symbol, exit_price, pnl, date, llm_client)  # mini LLM for post-mortem
    def detect_missed_opportunities(self, features, atrs, held_symbols, date)
    def maybe_reflect(self, trading_day_count, llm_client) -> bool  # returns True if reflected
    def get_beliefs_narrative(self) -> str
    def save(self) / def load(self)
```

## LLM Call Budget

| Type | Frequency | Calls/Year |
|------|-----------|-----------|
| Trading decisions | Hourly (every 4 bars) | ~1,500 |
| Post-mortems | Per closed trade (~10-30/yr) | ~20 |
| Weekly reflections | Every 5 trading days | ~50 |
| **Total** | | **~1,570** |

~5% overhead beyond existing calls.
